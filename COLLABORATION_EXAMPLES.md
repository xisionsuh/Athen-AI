# 여러 AI 협업 모드별 질문 예시

Athena AI는 질문의 복잡도와 특성에 따라 자동으로 최적의 협업 모드를 선택합니다.

## 협업 모드 종류

### 1. **Single (단일 AI)** - 기본 모드
- **설명**: 간단한 질문이나 특정 분야에 특화된 질문
- **사용 시기**: 빠른 응답이 필요한 단순한 질문
- **예시 질문**:
  - "안녕하세요"
  - "파이썬에서 리스트를 정렬하는 방법은?"
  - "오늘 날씨 어때?"
  - "간단한 요리 레시피 알려줘"

### 2. **Parallel (병렬 협업)** - 복잡한 질문
- **설명**: 여러 AI가 동시에 각자의 관점에서 답변을 생성하고, 총괄 AI가 종합
- **사용 시기**: 다양한 관점이 필요한 복잡한 질문
- **예시 질문**:
  - "인공지능이 미래 사회에 미칠 긍정적 영향과 부정적 영향을 종합적으로 분석해줘"
  - "블록체인 기술의 현재와 미래에 대해 기술적, 경제적, 사회적 관점에서 설명해줘"
  - "원격 근무의 장단점을 생산성, 워라밸, 커뮤니케이션 관점에서 분석해줘"
  - "기후 변화 해결을 위한 기술적 솔루션과 정책적 접근을 모두 제시해줘"

### 3. **Sequential (순차 실행)** - 심층 연구/보고서
- **설명**: 여러 AI가 순차적으로 작업을 분담하여 깊이 있는 연구나 보고서 작성
- **사용 시기**: 단계별로 나누어 처리해야 하는 복잡한 작업
- **예시 질문**:
  - "2024년 AI 기술 트렌드를 조사하고, 각 트렌드의 기술적 배경, 시장 영향, 미래 전망을 포함한 종합 보고서를 작성해줘"
  - "스타트업 창업 계획서를 작성해줘. 시장 분석 → 경쟁사 분석 → 비즈니스 모델 → 재무 계획 순서로"
  - "웹 애플리케이션 개발 프로젝트의 전체 아키텍처를 설계해줘. 요구사항 분석 → 시스템 설계 → 기술 스택 선택 → 데이터베이스 설계 순서로"

### 4. **Debate (토론 모드)** - 논쟁적 주제
- **설명**: 여러 AI가 서로 다른 입장에서 토론하고, 총괄 AI가 심판관 역할로 종합
- **사용 시기**: 논쟁적이거나 다양한 의견이 필요한 주제
- **예시 질문**:
  - "원자력 발전소를 확대해야 하는가, 축소해야 하는가? 각 입장의 근거를 제시하고 최종 결론을 내려줘"
  - "원격 근무가 오피스 근무보다 나은가? 양쪽 입장을 토론하고 결론을 내려줘"
  - "인공지능 개발에 대한 규제가 필요한가? 규제 찬성과 반대 입장을 모두 제시하고 판단해줘"
  - "암호화폐가 전통적인 금융 시스템을 대체할 수 있는가? 양쪽 관점에서 토론해줘"

### 5. **Voting (투표 모드)** - 선택지가 있는 결정
- **설명**: 여러 AI가 각각 의견과 선택을 제시하고, 다수결로 최종 결정
- **사용 시기**: 여러 선택지 중 하나를 선택해야 하는 경우
- **예시 질문**:
  - "스타트업을 위한 최적의 프로그래밍 언어는 무엇인가? Python, JavaScript, Go 중에서 선택해줘"
  - "웹 프레임워크 선택: React, Vue, Angular 중 어떤 것이 프로젝트에 가장 적합한가?"
  - "데이터베이스 선택: PostgreSQL, MongoDB, MySQL 중 어떤 것이 이 프로젝트에 적합한가?"
  - "클라우드 서비스 선택: AWS, Google Cloud, Azure 중 어떤 것이 비용 대비 효율적인가?"

## 실제 테스트 예시

### 병렬 협업 테스트
```
인공지능이 의료 분야에 미칠 영향에 대해 기술적, 윤리적, 경제적 관점에서 종합적으로 분석해줘
```

### 순차 실행 테스트
```
2024년 웹 개발 트렌드를 조사하고, 각 트렌드를 기술적 배경, 사용 사례, 학습 난이도, 미래 전망 순서로 정리한 보고서를 작성해줘
```

### 토론 모드 테스트
```
원자력 발전소를 확대해야 하는가, 재생 에너지로 전환해야 하는가? 각 입장의 근거를 제시하고 최종 결론을 내려줘
```

### 투표 모드 테스트
```
스타트업 MVP 개발을 위한 백엔드 프레임워크 선택: Node.js, Python (Django/FastAPI), Go 중에서 어떤 것이 가장 적합한가?
```

## 협업 모드 선택 기준

시스템은 다음 요소를 고려하여 자동으로 협업 모드를 선택합니다:

1. **복잡도 (Complexity)**
   - simple → single
   - moderate → single 또는 parallel
   - complex → parallel 또는 sequential
   - very_complex → sequential, debate, 또는 voting

2. **카테고리 (Category)**
   - conversation → single
   - technical → single 또는 parallel
   - creative → parallel
   - research → sequential
   - decision → debate 또는 voting

3. **질문의 특성**
   - 다양한 관점 필요 → parallel
   - 단계별 처리 필요 → sequential
   - 논쟁적 주제 → debate
   - 선택지가 있는 결정 → voting

## 참고사항

- 현재 스트리밍 모드는 **Single 모드만** 지원합니다
- 일반 채팅 API (`/api/chat`)는 모든 협업 모드를 지원합니다
- 각 협업 모드의 상세 의견은 채팅창 하단의 링크를 통해 확인할 수 있습니다
